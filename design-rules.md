# 디자인 규칙 (Design Rules)

> 마켓앱 UI/UX 설계 원칙과 디자인 시스템 규칙입니다.
> 당근마켓의 편의성과 토스의 간결함을 벤치마킹하여 적용합니다.

---

## 1. 디자인 철학

| 원칙 | 설명 |
|------|------|
| **간결함** | 화면 하나에 하나의 핵심 목적만 — 토스처럼 불필요한 요소 제거 |
| **신뢰감** | 데이터와 인증 배지를 통해 거래 신뢰성을 시각적으로 전달 |
| **접근성** | 누구나 직관적으로 사용할 수 있는 당근마켓식 친화적 UX |
| **일관성** | 색상, 폰트, 컴포넌트가 앱 전반에서 통일된 언어를 사용 |

---

## 2. 벤치마킹 레퍼런스

### 당근마켓에서 가져올 것
- 거래 카드 레이아웃 (이미지 + 제목 + 가격 + 위치 + 시간 구성)
- 직관적인 탭 네비게이션 구조 (홈 / 거래 / 커뮤니티 / 채팅 / 프로필)
- 목록 → 상세 → 채팅 시작으로 이어지는 심플한 거래 플로우

### 토스에서 가져올 것
- 숫자 데이터(시세, 가격)의 명확하고 큰 폰트 처리
- 불필요한 테두리, 배경 없이 여백으로 구분하는 화이트 스페이스 활용
- 행동 유도 버튼(CTA)의 명확한 위계와 색상 대비

---

## 3. 컬러 시스템

### 주요 색상 (추후 브랜드 색상 확정 시 업데이트)

```
Primary     #FF6B35   주황 계열 — 마켓 활성화, 주요 액션 버튼
Secondary   #1A1A2E   네이비 다크 — 신뢰감, 텍스트
Accent      #4ECDC4   민트 — 인증 배지, 성공 상태
Background  #F8F9FA   연한 회색 — 앱 기본 배경
Surface     #FFFFFF   흰색 — 카드, 모달 배경
```

### 상태 색상

```
Success   #28A745   거래 완료, 인증 성공
Warning   #FFC107   주의 필요, 임시 저장
Error     #DC3545   오류, 거래 취소
Info      #17A2B8   시세 정보, 안내 메시지
```

### 텍스트 색상

```
Primary Text    #1A1A2E   주요 텍스트 (제목, 가격)
Secondary Text  #6C757D   보조 텍스트 (날짜, 위치, 부가 정보)
Disabled Text   #ADB5BD   비활성 상태 텍스트
```

## 4. 타이포그래피

| 용도 | 크기 | 굵기 | 설명 |
|------|------|------|------|
| 화면 제목 | 22px | Bold (700) | 각 화면 상단 타이틀 |
| 섹션 제목 | 18px | SemiBold (600) | 카드 내 구분 제목 |
| 본문 | 15px | Regular (400) | 일반 텍스트, 설명 |
| 가격/시세 | 20px | Bold (700) | 숫자 강조 — 토스 스타일 |
| 보조 정보 | 13px | Regular (400) | 날짜, 위치, 조회수 |
| 버튼 텍스트 | 16px | SemiBold (600) | 주요 액션 버튼 |

- 기본 폰트: 시스템 폰트 (`System` - iOS: San Francisco, Android: Roboto)
- 숫자 데이터(가격, 시세)는 항상 강조 폰트 사이즈 적용

## 5. 컴포넌트 디자인 규칙

### 버튼
```
Primary 버튼   — 배경: Primary색, 텍스트: 흰색, 모서리: 12px, 높이: 52px
Secondary 버튼 — 배경: 투명, 테두리: Primary색 1px, 텍스트: Primary색
Danger 버튼   — 배경: Error색, 텍스트: 흰색 (삭제, 거래 취소 등)
```

### 카드 (거래 아이템 카드)
```
배경: Surface(흰색)
모서리: 12px
그림자: 가볍게 (elevation: 2, iOS shadow opacity: 0.08)
내부 여백: 16px
이미지 비율: 1:1 정방형, 모서리 8px
```

### 입력창 (Input)
```
배경: #F1F3F5
테두리: 없음 (포커스 시 하단 라인 또는 전체 테두리 Primary색)
높이: 52px
내부 좌우 여백: 16px
모서리: 10px
```

### 인증 배지 (Character Auth Badge)
```
배경: Accent색(민트) 반투명
텍스트: Accent색 진하게
아이콘: 체크 마크
위치: 유저 프로필 이미지 우하단 오버레이
```

## 6. 레이아웃 및 간격 시스템

### 기본 간격 단위 (4px 배수 사용)
```
xs:  4px   — 아이콘과 텍스트 사이
sm:  8px   — 관련 요소 간 소간격
md: 16px   — 기본 카드 내부 여백, 요소 간 표준 간격
lg: 24px   — 섹션 간 구분
xl: 32px   — 화면 상하단 여백
```

### 화면 레이아웃
- 좌우 여백: 최소 16px 유지
- 하단 탭바 영역 침범 금지 (SafeAreaView 적용 필수)
- 스크롤 가능한 콘텐츠 화면: 최하단 여백 80px 이상 확보 (탭바 가림 방지)

---

## 7. 아이콘 사용 규칙

- Expo에서 제공하는 `@expo/vector-icons` (MaterialIcons, Ionicons) 우선 사용
- 아이콘 크기: 탭바 24px / 본문 내 20px / 버튼 내 18px
- 커스텀 아이콘이 필요한 경우 SVG 형식으로 `assets/icons/`에 추가

---

## 8. 애니메이션 및 인터랙션

- 버튼 탭 시 `opacity` 0.7 처리로 피드백 제공 (`activeOpacity={0.7}`)
- 화면 전환: 기본 네이티브 슬라이드 사용 (과도한 커스텀 애니메이션 지양)
- 로딩 상태: 스켈레톤 UI 또는 `ActivityIndicator` 활용 (빈 화면 노출 금지)
- 토스트/스낵바: 하단 팝업 방식으로 오류/성공 피드백 제공

---

## 9. 접근성 원칙

- 색상만으로 상태를 구분하지 않을 것 (아이콘 또는 텍스트 병행)
- 터치 영역 최소 44x44px 보장 (iOS Human Interface Guidelines 기준)
- 중요한 버튼과 입력창에 `accessibilityLabel` 속성 작성
